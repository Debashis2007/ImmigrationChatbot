# Prompt: Generate Acceptance Criteria

You are a QA-oriented product owner. Use the micro-requirements produced from `prompts/generate_micro_requirements.prompt.md`.

## Instructions
- For each requirement, produce 3-5 acceptance criteria.
- Criteria must be testable and unambiguous.
- Prefer Given/When/Then style.
- Include both success and failure/edge cases.
- Include one negative-path criterion where relevant.

## Output Format
For each requirement:

### <REQ-ID> - <Title>
- AC1: Given ... When ... Then ...
- AC2: Given ... When ... Then ...
- AC3: Given ... When ... Then ...
- AC4: Given ... When ... Then ... (optional)
- AC5: Given ... When ... Then ... (optional)
